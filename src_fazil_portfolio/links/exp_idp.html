Role Overview

At IDP Education — one of the world's largest international student placement organizations — I managed production Oracle databases supporting the IELTS booking platform and student portal used by hundreds of thousands of students globally. The role required maintaining 99.9% uptime, ensuring data integrity, and coordinating software deployments with development teams.

A key part of this role was building automated shell-script validation pipelines that ran daily on remote Linux servers — checking database health, validating data integrity, and triggering alerts if anomalies were detected. This automation prevented manual toil and provided early warning of system issues before they affected users.

Software deployments were validated against IEEE and ISO software quality standards before being signed off — including documentation review, regression testing coordination, and change log maintenance.

Key Responsibilities

1

Automated Shell-Script Validation Pipelines

Wrote Bash scripts scheduled via cron (crontab -e) to run daily at 2 AM on the remote Linux server. Scripts performed: (a) row count checks across critical tables to detect unexpected deletions or bulk inserts; (b) null-check validation on mandatory fields; (c) foreign key integrity verification via SQL queries; (d) disk space threshold check with alert if <20% free. Any anomaly triggered an automated email alert to the DBA team via sendmail. This prevented 4+ potential data integrity incidents from reaching users.

2

System Health Monitoring & Alert Triggers

Implemented Oracle trigger functions that fired on critical table modifications — logging the change, user, timestamp, and old/new values to an audit table. Separate shell script monitored the Oracle alert log (alert_SID.log) for ORA- errors and paged the on-call DBA if critical errors appeared. Average time-to-alert: <5 minutes from incident to notification.

3

Database Performance Tuning

Used Oracle AWR (Automatic Workload Repository) reports to identify top SQL statements by elapsed time. Analyzed execution plans with EXPLAIN PLAN and DBMS_XPLAN. Added covering indexes on high-traffic query predicates — reduced average query time on the student search endpoint from 2.3s to 0.4s. Identified and resolved index fragmentation on the booking table (1.2M+ rows).

4

Software Deployment Validation — IEEE/ISO Standards

Coordinated with the software development team on production release cycles for the IELTS booking platform and student portal. Validated each release against an IEEE-based software verification checklist: code review sign-off, regression test pass report, database schema migration validation (no breaking changes), rollback plan documented. Maintained ISO-aligned change log documentation for all production changes — required for audit compliance.

5

Backup & Disaster Recovery

Managed Oracle RMAN backup strategy: daily incremental backups + weekly full backup to separate storage. Tested recovery procedures quarterly by restoring to a staging environment and verifying data integrity. Documented RTO (Recovery Time Objective): <4 hours. RPO (Recovery Point Objective): <24 hours. Participated in one actual recovery drill — successfully restored full database within 2.5 hours.

6

Cross-Functional Collaboration

Worked directly with the software development team to review schema changes before deployment, validate SQL queries in code PRs for performance, and provide production database statistics for capacity planning. Documented schemas, data dictionaries, and maintenance procedures — enabling any team member to operate the database independently in emergency situations.
