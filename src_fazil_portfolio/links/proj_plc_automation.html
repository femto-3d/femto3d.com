Project Overview

A comprehensive study and implementation of industrial PLC programming using Allen-Bradley PLCs and RSLogix Studio 5000 IDE. Projects Built — Step by Step

1

Conveyor Belt Control System — Multi-Zone Management

Designed a PLC program to control a 3-zone industrial conveyor with start/stop pushbuttons, variable speed control output (0–100% PWM to VFD), and automatic jam detection via load current sensors on each zone. Implemented timers for belt acceleration/deceleration ramps (5-second soft start to prevent product spillage). Configured solenoid valve output for pneumatic brake release. Emergency Stop (E-STOP) circuit follows IEC 62061 PLd (Performance Level d) standards — pressing E-STOP immediately de-energizes all motor contacts and applies the pneumatic brake within 500ms. Used ladder logic blocks: timers (TON/TOF), compare instructions for speed setpoint logic, and a shift register to track product passage across zones. HMI displays real-time belt speed, load current per zone, and jam alerts with zone identification.

2

Motor Start/Stop Sequencing — Multi-Motor Synchronization

Developed a PLC control program for a three-motor system (pump, compressor, post-process fan) with mandatory sequential startup. When the "Start" button is pressed in AUTO mode, Motor 1 (pump) runs for 2 seconds before Motor 2 (compressor) is energized; Motor 3 (fan) energizes only after Motor 2 has been stable for 3 seconds. This startup sequence prevents inrush current spikes and cascading pressure surges. Implemented manual mode where each motor can be individually controlled via discrete pushbuttons. Fault detection: each motor has an overload current sensor; if current exceeds threshold for >2 seconds, the motor is de-energized and a fault indicator lights on HMI. Recovery requires operator acknowledgment (press "Fault Reset" button) and manual motor restart. Ladder logic uses interlocked timer blocks to enforce the 2–3 second delays.

3

Safety Interlock System — Light Curtain Integration (ISO 13849)

Implemented a safety-certified interlock program for a guarded machine access area using a light curtain (photoelectric safety sensor). The control logic follows ISO 13849-1 PLr (Performance Level "c") standards. When the light curtain beam is broken (intruder in guarded zone), the PLC immediately commands: (1) stop all moving axes within 200ms, (2) actuate safety solenoid to lock access gate, (3) illuminate red warning beacon, (4) log the intrusion event with timestamp. Light curtain signal is wired to the PLC's safety input module (redundant dual-channel contact). Re-enabling machine operation requires both: (1) light curtain beam restoration (clear zone), AND (2) operator pressing a mandatory reset button. Used safety-rated contacts (series redundancy) in the E-STOP circuit to meet PLr certification. Documented all interlock logic with cross-references to ISO 13849 safety categories.

4

Pneumatic Actuator Control — Double-Acting Cylinder Sequencing

Programmed PLC control for double-acting pneumatic cylinders with solenoid valve actuation and end-of-stroke confirmation. The system controls a 4-cylinder assembly operation: (1) extend cylinder A (solenoid A energized), wait for "fully extended" proximity sensor feedback, (2) extend cylinder B (while A held), confirm with sensor, (3) retract B, confirm sensor, (4) retract A, confirm sensor. Each cylinder stage has a 5-second timeout — if expected sensor signal doesn't arrive within timeout, the system aborts, logs a fault, and returns all cylinders to the home position. Used ladder logic with latched outputs for solenoid control, timer-based timeout logic, and state machine rungs to enforce strict sequencing. Implemented manual jog controls (override mode) for setup/maintenance where operators can manually extend/retract each cylinder while monitoring sensor feedback on the HMI.

5

Process Timer/Counter Logic — Batch Production & Tracking

Designed a batch counting and product tracking system using PLC counter blocks and shift registers. When a "Product Present" sensor detects an item at the conveyor entry, a counter is incremented. Simultaneously, a shift register stores the product count at each of 5 downstream stations (assembly, test, packaging, QC, shipping). At each station, when the product count reaches a predefined batch size (e.g., 50 units), the PLC: (1) logs the batch number, (2) signals the solenoid to eject the full batch into a collection bin, (3) resets the counter, (4) starts a new batch cycle. The shift register allows the PLC to track which batch is currently at each station — enabling real-time production timeline visualization on the HMI (e.g., "Batch #127 is currently in assembly, Batch #126 in QC, Batch #125 in shipping"). Implemented with compare blocks to detect batch boundaries and one-shot triggers to prevent double-counting.

6

HMI Design & Operator Interface — RSView

Built an intuitive HMI dashboard in RSView (Rockwell's SCADA software) that displays: real-time status indicators (running/stopped/fault), animated process visualization (conveyor belt speed, motor states, solenoid actuation), fault message log with timestamps, production count display, and manual control buttons (start, stop, emergency stop, fault reset, manual jog). Implemented color-coded status: green (running), yellow (caution/slow), red (fault/stopped). The HMI connects to the PLC via Ethernet and updates every 500ms. Designed the interface to match industrial ergonomics standards — large buttons for safety-critical controls, clear font sizes for readability from 2+ meters distance. Tested the HMI with simulated PLC faults to verify alarm display and operator response paths.

7

Simulation, Validation & Documentation

Simulated all PLC programs in RSLogix Studio 5000 Logix Designer before deployment. Used the built-in simulator to inject test inputs (button presses, sensor transitions) and verify outputs (solenoid actuation, motor commands) respond correctly. Tested edge cases: E-STOP during various operation states, sensor failures (missing expected feedback), intentional faults (overcurrent), and recovery sequences. For each program, created detailed documentation: (1) I/O tag list (all inputs and outputs with descriptions), (2) ladder diagram printouts with rung comments, (3) safety analysis per ISO 13849, (4) operation manual for technicians. All programs passed 100-hour continuous simulation without lockups or unexpected behavior.
