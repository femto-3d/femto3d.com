Project Overview

A Raspberry Pi-based home security system featuring real-time motion detection via PIR sensors, automatic image capture, immediate email alerts with attached photos, and live network video streaming. The system demonstrates practical Embedded Linux programming, GPIO interrupt handling, SMTP integration, and real-time camera control.

When motion is detected, the system automatically captures a high-resolution photo and a 10-second video clip to local storage, then dispatches an email alert with the photo attachment within 3 seconds. A Flask-based web server broadcasts a live MJPEG video stream accessible from any browser on the home network for remote surveillance monitoring.

Key features: configurable detection zones, active hours scheduling, 4-hour continuous operation on battery, and sub-3-second alert dispatch latency. Demonstrates production-grade embedded security design patterns.

How It Was Built — Step by Step

1

PIR Motion Detection — GPIO Interrupt Handling

Wired the HC-SR501 PIR sensor to Raspberry Pi GPIO pin 17 (BCM numbering). Configured the GPIO as an input with pull-down resistor. Used Python RPi.GPIO library to register a rising-edge interrupt handler that triggers when motion is detected. The interrupt callback function timestamps the event and sets a motion flag. Implemented a 5-second debounce lockout to prevent false trigger cascades (PIR modules often output multiple pulses per detection event). Tested sensitivity adjustment by tuning the potentiometer on the PIR module — configured for ~3-meter detection range at optimal indoor sensitivity.

2

Image Capture — Raspberry Pi Camera Module v2

Connected Raspberry Pi Camera Module v2 via the ribbon cable to the CSI connector. Used the Python picamera library to capture images on-demand. When motion is detected (via GPIO interrupt), the system captures: (1) a single 1080p JPEG image (typically 350KB) and (2) a 10-second H.264 video clip at 720p 30fps. Both files are timestamped and saved to /home/pi/security_footage/ directory. Implemented automatic image rotation detection for orientation-independent security footage. Tested capture latency: 0.8 seconds from motion trigger to first photo on disk.

3

Email Alert System — SMTP via Gmail

Implemented Python SMTP client using the smtplib library to send alerts via Gmail's SMTP server (smtp.gmail.com:587 with TLS encryption). Each alert email includes: timestamp of motion event, zone label (configurable, e.g., "Front Door", "Garage"), and the captured JPEG image as an email attachment. Generated an app-specific password for Gmail account (not the standard password) for security. Email dispatch happens asynchronously in a background thread to avoid blocking the main motion detection loop. Measured latency: motion trigger at T=0 → email sent by T=2.8 seconds (on average). Tested with 20+ alerts over 24 hours — 100% delivery rate to inbox (no spam folder).

4

Live MJPEG Video Stream — Flask Web Server

Developed a minimal Flask web application that streams live camera frames as MJPEG (Motion JPEG — individual JPEG frames sent with HTTP multipart boundaries). The camera captures frames at 15 fps (configurable), encodes each frame to JPEG at 80% quality, and streams to connected browsers via HTTP. Implemented variable frame rate control to manage bandwidth — lower frame rates on slower networks. Accessible via http://192.168.1.50:5000/video_feed on the local network. Tested simultaneous viewing from 3 devices without frame drops. Mobile-responsive HTML interface displays the video feed with manual control buttons (stop, pause, restart stream).

5

Detection Zones & Active Hours Scheduling

Created a configuration file (JSON format) that allows users to define: (1) detection zones (front door, garage, backyard) with custom labels, (2) active hours (e.g., 8 PM – 7 AM when away), (3) PIR sensitivity level (1–10 scale mapped to detection threshold), (4) alert recipients (multiple email addresses). The Python main loop reads this config on startup and during operation. If current time falls outside active hours, PIR triggers are ignored (silent mode). Implemented a simple web API endpoint where users can update config without SSH'ing into the Pi — changes take effect within 5 seconds.

6

Storage Management — Automatic Cleanup

Implemented a background Python script that runs hourly via cron to clean up old security footage. Rule: any image or video file older than 7 days is deleted. This prevents the Raspberry Pi's SD card from filling up over time. Monitored storage: at 2 motion events per day × 1.5 MB per event, the system uses ~21 MB/week, requiring cleanup roughly every 4 months for a 32 GB SD card. Implemented warning system: if free disk space drops below 200 MB, the system logs a warning and disables new capture (preventing corrupted files from writing to a full disk).

7

Integration, Testing & Deployment

Integrated all components into a single Python application with daemon process management (systemd service unit). Tested the complete pipeline: (1) trigger PIR manually → verify interrupt fires, (2) confirm image captured within 1 second, (3) verify email arrives within 3 seconds with attachment, (4) confirm live stream accessible from browser. Conducted 72-hour continuous operation test: 15 motion events across 3 days, 100% email delivery, zero crashes, zero frame drops on live stream. Deployed systemd service that auto-starts on Pi reboot. Tested battery operation on a 5V/10Ah power bank — system ran for 4+ hours at 30% CPU utilization, 70% idle time (motion detection is low-power).
