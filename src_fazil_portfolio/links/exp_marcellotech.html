Role Overview

At Marcellotech — an engineering education and training institute in Tiruchirappalli — I designed and delivered a structured curriculum on Embedded Systems, IoT, and Robotics for engineering students. This role required not just deep technical knowledge, but also the ability to communicate complex embedded concepts clearly and design hands-on lab exercises that build real intuition.

I mentored 15+ students through independent project work — guiding them from bare-metal blinking LEDs to complete IoT-connected automation systems with sensors, actuators, and cloud dashboards. Several students from my batches went on to win college and state-level competitions using projects developed under my guidance.

This role also deepened my own understanding — being required to explain interrupt service routines, DMA, timer configuration, I2C arbitration, and PID tuning at a whiteboard level forced me to build an exceptionally thorough mental model of embedded systems architecture.

Curriculum & Teaching Topics

1

Microcontroller Architecture — AVR & ARM Cortex

Taught the Harvard vs Von Neumann memory architecture distinction, register file organization, pipeline stages, interrupt vector tables, and peripheral bus hierarchy for AVR (ATmega328P) and ARM Cortex-M (STM32) processors. Students learned to read datasheets and configure peripherals (timers, ADC, UART) via direct register manipulation — building true hardware-level understanding rather than relying on abstraction libraries.

2

Embedded C Programming

Structured Embedded C course: bit manipulation, memory-mapped I/O, volatile variables, ISRs (interrupt service routines), DMA basics, state machines, and circular buffer implementation. Lab exercises: timer-based LED blink, UART echo, ADC sampling loop, PWM servo control, SPI accelerometer read. Students progressed from blinking LEDs to interrupt-driven sensor systems in 8 weeks.

3

Sensor Interfacing — I2C, SPI, UART

Hands-on labs for all three serial protocols: I2C (MPU-6050 IMU, OLED display, BMP280 pressure sensor), SPI (SD card, ADXL345 accelerometer), UART (GPS module, HC-05 Bluetooth, Nextion display). Students learned to use logic analyzers to debug protocol-level issues — distinguishing ACK errors, clock stretching, and framing errors by waveform inspection.

4

IoT System Integration — ESP32 & MQTT

Project-based IoT module: ESP32 MicroPython/Arduino programming, Wi-Fi provisioning, MQTT publish/subscribe, cloud dashboard integration (ThingSpeak, IFTTT). Final project: students built a complete IoT monitoring system — sensor node + gateway + dashboard — within the course timeframe. Topics covered: JSON payloads, QoS levels, OTA updates, and power management for battery-powered nodes.

5

Motor Control & Robotics Applications

Servo, DC, and stepper motor control theory and practice. H-bridge driver (L298N, L293D) operation, PWM frequency selection, back-EMF considerations. Basic PID control theory and implementation for line-following robots. Capstone: students built a 2-wheeled self-balancing robot or line-following car using everything learned in the course.
