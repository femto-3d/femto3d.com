Project Overview

A production multi-agent AI system designed and deployed for Crestlane Group to automate business workflows including: inbound query classification, intelligent email response drafting, marketing content generation, and analytics summarization. The system leverages both OpenAI GPT-4 and Claude (Anthropic) LLMs with specialized agents for each task, all orchestrated through N8N workflow automation on a dedicated Linux server.

The architecture uses isolated Docker containers for each agent to prevent cross-contamination and hallucination bleed-through. N8N serves as the central orchestrator, handling webhook triggers, message routing, human review queues, and result delivery via email/API. The entire system is version-controlled via Git with CI/CD pipeline for zero-downtime deployments.

Key achievement: Deployed to production with 24/7 uptime, processing 500+ client outreach emails per week with 35% higher open rate compared to template-based emails. Demonstrates ability to architect and deploy enterprise-grade AI automation systems.

How It Was Built — Step by Step

1

System Architecture — Multi-Agent Design Pattern

Architected a 4-agent microservices system where each agent handles a specific business function: (1) Agent Classifier — analyzes inbound queries and routes to appropriate handler, (2) Agent Drafter — writes personalized email responses, (3) Agent Content — generates marketing copy and promotional material, (4) Agent Analytics — summarizes performance metrics and trends. Each agent is a separate Docker container with its own isolated environment and memory state (no shared state between agents). This isolation prevents hallucinations in one agent from corrupting another agent's logic. Communication between agents happens only through N8N message queues (Redis) — ensuring clean interfaces and preventing tightly-coupled dependencies.

2

LLM Integration — OpenAI GPT & Claude Anthropic

Integrated both OpenAI GPT-4 and Claude (Anthropic) LLMs with specialized system prompts for each agent role. Agent Classifier uses GPT-4 for structured JSON output (category, confidence score, routing rule). Agent Drafter uses Claude for longer-form content generation with better writing quality. Agent Content uses GPT-4 for creative content and marketing angles. Agent Analytics uses Claude for summarization and trend detection. All API keys stored securely in environment variables (no hardcoded secrets). Implemented exponential backoff retry logic for API failures — if OpenAI returns rate limit error, wait 60 seconds then retry (max 3 attempts). Measured API latencies: GPT-4 ~2–3 seconds per request, Claude ~3–5 seconds depending on output length.

3

N8N Workflow Orchestration — Visual Workflow Automation

Built the central orchestration logic in N8N (low-code workflow automation platform). Workflows are visual, non-code blocks connected by arrows — making it auditable and maintainable without deep engineering knowledge. Main workflow: (1) Webhook receives inbound email, (2) HTTP request to Agent Classifier (returns JSON category), (3) Conditional routing based on category, (4) Call appropriate agent (Drafter/Content/Analytics), (5) Format response, (6) Send to human review queue (database table), (7) Await human approval, (8) Deliver via SMTP email. Implemented branching for different email types (lead inquiry, complaint, partnership request) with different handlers. All N8N workflows are version-controlled as JSON in Git — enabling rollback and change tracking.

4

Docker Containerization — Isolated Microservices

Containerized each agent in its own Docker image built from a Python 3.11 base. Each container includes: (1) minimal Python dependencies (requests, openai library, anthropic library, redis client), (2) entry point script that connects to Redis message queue, (3) graceful shutdown handlers. docker-compose.yml defines the multi-container setup: 4 agent containers, 1 N8N container, 1 Redis container, 1 PostgreSQL database (for workflow persistence and review queue). All containers run on a shared Docker network (service discovery via container names). Implemented health checks: each agent container exposes a /health HTTP endpoint that returns {"status": "healthy"} if the container can connect to Redis and reach LLM APIs. Docker Compose monitors health and auto-restarts failed containers. Resource limits: each agent container capped at 512MB RAM, 1 CPU core to prevent runaway processes.

5

Client Outreach Automation — Lead List Processing

Implemented a workflow that polls a lead database (PostgreSQL) daily and drafts personalized outreach emails for new leads. The flow: (1) N8N scheduled trigger (8 AM daily), (2) Query database for leads added in past 24 hours, (3) For each lead, Agent Drafter generates a personalized email using lead properties (name, company, industry) as context variables, (4) Queue the email in review table, (5) Notify Crestlane manager via webhook to review/approve. Manager can click "approve" in email — N8N webhook receives approval, (6) Email sent via SMTP to lead. Measured: 500+ emails per week, 6-minute average time from lead entry to approval prompt, 95% approval rate from manager (indicating AI quality is trusted). Email open rate: 42% for AI-drafted emails vs 31% for template emails (35% improvement).

6

CI/CD Pipeline — Git-Based Deployment

Set up Git repository with all code, Docker configs, and N8N workflow definitions. Implemented GitHub Actions CI/CD pipeline: (1) on push to main branch, run linting and unit tests for agent code, (2) if tests pass, build Docker images and push to container registry, (3) SSH into production server, (4) git pull latest code, (5) docker-compose up --build (causes zero-downtime rolling restart — N8N waits for agents to healthcheck before routing new messages), (6) verify all containers healthy, (7) run smoke tests (send test email through system), (8) if all pass, deployment is complete. Rollback: git revert to previous commit, push — pipeline automatically rolls back all containers. Time from code commit to production: ~5 minutes with full test coverage.

7

Monitoring, Analytics & Continuous Improvement

Implemented comprehensive logging: every LLM API call logged with input tokens, output tokens, latency, and cost. N8N logs every workflow execution (webhook received, which branch taken, approval status, final delivery). All logs shipped to a centralized logging service (ELK stack on the same server). Created dashboards: (1) email volume (inbound vs outbound), (2) agent latency distribution (min/avg/max), (3) approval rates by email category, (4) LLM API spend per week, (5) error rates by agent. Agent Analytics generates a weekly report: email open rate, response rate, categorization accuracy (vs. human validation sample), cost per email. Data shows system is profitable: average cost per outreach email (LLM + infrastructure) = $0.47, but each conversion is worth $500+ to Crestlane, so ROI is 1000%+.
