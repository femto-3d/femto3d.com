Role Overview

At Crestlane Group — a digital marketing and client services company — I designed and deployed a production-grade multi-agent AI automation system that handles client outreach, marketing campaigns, intelligent email replies, and business analytics with minimal human intervention.

The system uses OpenAI GPT and Claude (Anthropic) as the core reasoning engines, N8N as the orchestration and workflow layer, and Docker for agent isolation and deployment. A dedicated Linux server hosts the full stack, with Git-based CI/CD pipelines for continuous deployment of workflow and prompt updates.

This role required full-stack engineering skills: system architecture design, Docker infrastructure, API integration, Python scripting, server administration, and prompt engineering — all applied to building a reliable, production-safe AI automation system.

Key Contributions

1

Multi-Agent AI Architecture Design

Designed a multi-agent system with 4 specialized AI agents: (A) inbound query classifier, (B) email/message response drafter, (C) marketing content generator, (D) business analytics summarizer. Each agent was given a tightly-scoped system prompt with explicit output format requirements. Agents operate independently and communicate only through structured JSON messages via a shared Redis queue — no shared conversational memory, preventing cross-agent hallucination.

2

Docker Containerization & Server Infrastructure

Each AI agent runs in its own Docker container, defined in a docker-compose.yml file with resource limits (CPU, memory) to prevent one runaway agent from starving others. Containers share only the Redis message broker. A dedicated Linux server (Ubuntu 22.04) hosts the full stack. Nginx reverse proxy routes external webhook requests to the correct N8N endpoint. Systemd and Docker restart policies ensure 24/7 uptime with auto-restart on crash.

3

N8N Agentic Workflow Orchestration

Configured N8N (self-hosted) as the visual workflow engine connecting triggers (email webhooks, scheduled crons, form submissions) to AI agents and output channels (SMTP, CRM, Slack). Workflows include: inbound email → classify → route to drafter agent → output to approval queue → send. All workflows are version-controlled via N8N's export JSON, committed to Git for auditability and rollback capability.

4

CI/CD Pipeline for Deployment

Implemented a Git-based CI/CD pipeline using GitHub Actions: code push to main branch → GitHub Actions workflow validates JSON/Python syntax → SSH into server → docker-compose pull && docker-compose up --build -d → health-check endpoint confirms all containers running → notification sent. Zero-downtime rolling deployment for prompt and workflow updates.

5

Client Outreach & Marketing Automation

Built an outreach pipeline: N8N polls the lead database (Google Sheets / Airtable) → formats each lead's context → GPT-4 drafts a personalized outreach email (name, business type, specific pain point) → SMTP sends → reply webhook detects responses → classifies intent → routes to follow-up agent. Weekly analytics report (open rate, reply rate, conversion) auto-generated by the Analytics Agent and emailed to the business owner.

6

Web Design, Deployment & Client Dashboards

Designed and deployed client-facing web dashboards for campaign performance analytics. Built with HTML/CSS/JavaScript, deployed via Git push to Netlify/server. CI/CD pipeline handles automatic rebuild on commit. Dashboards pull live data from N8N via REST API and render charts using Chart.js. Clients access their own branded dashboard to view outreach metrics, response rates, and campaign ROI.
